# Soft Wellness

Красивый mobile-first PWA-дневник питания с локальным хранением данных и развёртыванием через Cloudflare Workers + Wrangler.

## Что работает

- Первое приветствие с отдельными инструкциями установки для iPhone и Android.
- Анкета пользователя: имя, возраст, рост и вес.
- Автоматический расчёт целей:
  - калории = вес × 27;
  - белок = вес × 2;
  - жиры = вес × 0,8;
  - углеводы = (калории − белок × 4 − жиры × 9) ÷ 4.
- Дневник по датам: завтрак, обед, ужин, перекус.
- Каталог продуктов, поиск, граммовка, автоматический пересчёт КБЖУ.
- Создание своих продуктов.
- Изменение и удаление записей.
- Поиск продукта по штрихкоду через Open Food Facts и Cloudflare Worker.
- История веса, график, средние показатели и серия ведения дневника.
- Экспорт и импорт JSON-резервной копии.
- Локальное хранение в браузере без серверной базы данных.
- PWA manifest + Service Worker для установки и офлайн-запуска.
- Базовая адаптация под Telegram Mini App.

## Локальный запуск

Нужен Node.js 20+.

```bash
npm install
npm run dev
```

Wrangler покажет локальный адрес, обычно `http://localhost:8787`.

## Проверка перед публикацией

```bash
npm run check
```

## Ручная публикация в Cloudflare

```bash
npx wrangler login
npm run deploy
```

После первой публикации Wrangler создаст Worker `soft-wellness` и выдаст HTTPS-адрес на `workers.dev`.

## Автопубликация из GitHub

В репозитории уже лежит `.github/workflows/deploy.yml`.

В GitHub откройте:

`Settings → Secrets and variables → Actions → New repository secret`

Добавьте ровно две секретные переменные:

1. `CLOUDFLARE_ACCOUNT_ID` — идентификатор аккаунта Cloudflare.
2. `CLOUDFLARE_API_TOKEN` — API Token с разрешением **Edit Cloudflare Workers** только для нужного аккаунта.

После каждого push в ветку `main` GitHub Actions выполнит `wrangler deploy`.

## Приватность

Профиль, дневник, пользовательские продукты и история веса хранятся в `localStorage` конкретного браузера/веб-приложения. Очистка данных сайта удалит дневник, поэтому в профиле предусмотрен экспорт резервной копии.

## Источники и вдохновение

Дизайн — оригинальная реализация по выбранному Soft Wellness-концепту. Архитектурные принципы локальной приватности, быстрого ежедневного потока и «настроить один раз — отмечать в несколько касаний» вдохновлены проектом Aerko_ автора SrPakura. Код Aerko_ в этот репозиторий не копировался.

Публичная база штрихкодов: Open Food Facts.
