# Soft Wellness 1.4 — исправление Telegram-сессии

## Что исправлено

- один Telegram-пользователь теперь может быть связан только с одной активной PWA-сессией;
- старые дублирующиеся тестовые привязки автоматически отключаются при новом `/start КОД`;
- кнопка в сообщении бота содержит временный билет именно текущей PWA-сессии;
- билет дополнительно проверяется по подтверждённому `Telegram.WebApp.initData`;
- Mini App получает серверный снимок до показа дневника, поэтому не успевает показать отдельный локальный профиль;
- добавлен ID общей сессии в экран Telegram-настроек для диагностики;
- PWA и Mini App подтягивают удалённые изменения при фокусе/возврате и каждые 12 секунд.

## Порядок переподключения после обновления

1. Разверните проект.
2. Откройте PWA и обновите её через Safari/Chrome, чтобы установился новый Service Worker.
3. В профиле откройте Telegram-нагадування и нажмите **Новый код**.
4. В боте отправьте `/start НОВЫЙ_КОД`.
5. Открывайте Mini App именно кнопкой **«Відкрити ту саму сесію»**, которую бот пришлёт после привязки.
6. Сравните первые 8 символов ID общей сессии в PWA и Mini App — они должны совпадать.

## Секрет бота

`TELEGRAM_BOT_TOKEN` должен храниться только как encrypted secret Cloudflare/GitHub Actions. Не помещайте реальный токен в репозиторий, `.dev.vars.example`, JavaScript или ZIP.
