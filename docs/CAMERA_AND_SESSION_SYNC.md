# Камера та синхронізація сесії

## Камера

Сканер спочатку пробує нативний Barcode Detection API. Якщо він відсутній або не підтримує лінійні коди, використовується `@zxing/browser` 0.2.1 через закріплений CDN URL. Є резервне сканування з фотографії.

Перевірте:

- сайт відкритий через HTTPS;
- у Safari або Telegram дозволено доступ до камери;
- камера не зайнята іншим застосунком;
- штрихкод займає більшу частину кадру, без відблисків.

## Telegram Mini App

Після підключення бот надсилає кнопку Web App з URL поточного Worker. Mini App викликає `/api/telegram/session`, передаючи `Telegram.WebApp.initData`. Worker:

1. перевіряє HMAC підпис;
2. знаходить користувача за Telegram ID;
3. видає додатковий токен на ту ж сесію;
4. повертає стан через `/api/sync/state`.

Нові endpoints:

- `POST /api/telegram/session`
- `GET /api/sync/state`
- `PUT /api/sync/state`

`Durable Object` зберігає до шести активних токенів однієї сесії, тому токен PWA не анулюється при відкритті Mini App.


## Исправление общей сессии в 1.4

Причина разных сессий была в старых повторных привязках: несколько PWA-профилей могли хранить один и тот же Telegram chat id, а поиск выбирал не обязательно последний профиль.

Теперь Worker:

1. хранит однозначный ключ `chat:<telegram_user_id> -> pwa_session_id`;
2. при новом `/start КОД` отключает предыдущую привязку этого Telegram-пользователя;
3. создаёт временный `launch ticket`, привязанный одновременно к Telegram user id и PWA session id;
4. передаёт ticket в URL кнопки Web App;
5. возвращает облачное состояние прямо в ответе `/api/telegram/session`;
6. клиент блокирует интерфейс до применения этого состояния;
7. при возврате в PWA или Mini App повторно подтягивает изменения и проверяет обновления каждые 12 секунд, пока приложение открыто.
