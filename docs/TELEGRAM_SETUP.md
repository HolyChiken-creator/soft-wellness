# Telegram-first налаштування

## Для користувача

- `/start` — створити або відкрити постійний Telegram-акаунт Soft Wellness і отримати код для PWA;
- `/code` — отримати новий одноразовий код входу для PWA;
- `/status` — перевірити сесію й нагадування;
- `/pause` — призупинити нагадування;
- `/resume` — відновити нагадування.

Mini App входить автоматично через перевірений `Telegram.WebApp.initData`. Код для Mini App не потрібен.

PWA входить через одноразовий восьмисимвольний код, створений ботом. Код живе 10 хвилин і видаляється після першого успішного обміну.

## Синхронізація даних

Профіль, щоденник, вода, вага та налаштування зберігаються у Durable Object під одним внутрішнім session ID, який однозначно пов'язаний з Telegram user ID. `localStorage` Safari PWA і Telegram WebView не спільний, тому обидва клієнти синхронізуються з Worker.

При першому підключенні PWA до вже заповненої Telegram-сесії користувач сам обирає джерело даних. Це запобігає непомітному перезапису щоденника.

## BotFather

Встановіть Main Mini App і домен Worker. GitHub Actions також налаштовує menu button на URL задеплоєного Worker.
